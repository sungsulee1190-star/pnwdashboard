## data.js 데이터 구조 (현재 전체)

| 변수명 | 내용 | 행수 |
|--------|------|------|
| `DATA_EXPORT` | 중국산 국가별 월별 수출량 | ~10,000행 |
| `DATA_KR_EXPORT` | 한국산 회사별 월별 합계 | ~300행 |
| `DATA_KR_REGION` | 한국산 회사+지역별 월별 | ~3,000행 |
| `DATA_KR_YEARLY` | 한국산 회사별 연간 합계 | 30행 |
| `DATA_KR_TOTAL` | 한국산 전체 연간 합계 | 9행 |
| `DATA_KR_COUNTRY` | 한국산 국가별 (현재 빈 배열) | 0 → update_data.py 실행 후 채워짐 |
| `DATA_PRICE` | 중국 내수가 (CNY/톤) + USD 환율 | JSON 객체 |
| `DATA_CAPA` | 공장별 CAPA (45개) | 배열 |
| `DATA_COMPANIES` | 경쟁사 프로필 (14개) | 배열 |
| `DATA_INDICATORS` | 중국 코트지 생산/수출/내수/재고 월별 | ~33행 |
| `DATA_NEWS` | 뉴스피드 | JSON 객체 |

## 월별 업데이트 절차

```
① 엑셀 파일 받기 (한국수출실적, 중국수출실적, 중국아트지가격)
② Excel에서 열고 → F12 다른 이름으로 저장 (DRM 해제)
③ python update_data.py
④ git add data.js && git commit -m "data: YYYY년 MM월" && git push
```

## 개선 권고사항 (우선순위순)

### 🔴 즉시 개선 필요

| # | 항목 | 이유 |
|---|------|------|
| 1 | **엑셀 DRM 자동 해제 워크플로우** | 매달 수동 "다른 이름으로 저장" 필요. 회사 IT팀에 DRM 예외 폴더 설정 요청 권장 |
| 2 | **DATA_KR_COUNTRY 채우기** | 한국수출실적 엑셀에 국가 컬럼이 있으면 즉시 가능. DRM 해제 후 update_data.py 실행 |
| 3 | **중국수출실적 2025~2026 데이터** | 현재 data.js에 2022~2024 데이터만. 신규 파일 연결 시 최신 데이터 반영 |

### 🟡 중기 개선

| # | 항목 | 효과 |
|---|------|------|
| 4 | **수출실적 탭에 수출지도 추가** | 구버전에 있던 `mapTrade` (국가별 원형 버블맵) — 시각적 임팩트 큼 |
| 5 | **중국 주요지표 차트 (생산/수출/내수/재고)** | 구버전에 있던 `cInd` 차트 — 현재 중국산 수출실적 탭에 없음 |
| 6 | **경쟁사 재무 데이터 추가** | 현재 `DATA_COMPANIES.financials` 비어있음 → 분기 매출/영업이익 입력 |
| 7 | **뉴스피드 실데이터** | 현재 빈 샘플 데이터 → 업계 뉴스 수동 입력 or 크롤링 연동 |
| 8 | **가격동향에 수출가 라인 추가** | 내수가 + 수출 FOB 가격 비교 → 한국산 경쟁력 직관적 파악 |
